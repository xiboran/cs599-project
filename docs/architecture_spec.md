# 研究生智能学习助手 - 架构设计说明书

## 1. 整体架构

采用分层模块化架构，从下到上分为数据层、工具层、核心能力层、业务逻辑层、表现层五层。

### 1.1 架构分层

- **数据层**：环境配置文件、ChromaDB向量库、JSON用户画像持久化
- **工具层**：课程信息、考试信息、作业状态、学习建议、知识库检索五类工具函数
- **核心能力层**：单Agent工具调用、多Agent路由编排、RAG检索、双轨记忆系统
- **业务逻辑层**：对话流程编排、画像自动更新、特殊指令处理
- **表现层**：Streamlit Web交互界面

## 2. 技术栈选型

| 层级 | 技术选型 | 作用 |
|------|----------|------|
| 大模型 | 通义千问 qwen-turbo | 核心推理与工具调用 |
| Agent框架 | LangChain + LangGraph | 工具调用编排、多Agent状态流转 |
| 向量数据库 | ChromaDB | 知识库向量存储与相似度检索 |
| 前端框架 | Streamlit | 快速构建Web交互界面 |
| 文档处理 | PyPDF + langchain-text-splitters | PDF加载与文本分割 |
| 记忆存储 | 内存 + JSON文件 | 短期对话记忆与长期画像持久化 |

## 3. 核心模块设计

### 3.1 单Agent模块（LearningAgent）

基于LangGraph实现ReAct模式，支持5个工具的自动调用与结果生成。

### 3.2 多Agent模块（StudyGraph）

包含路由Agent、课程Agent、论文Agent、规划Agent，通过条件边实现问题分发与结果汇总。

### 3.3 RAG检索模块

采用「文档加载→文本切分→向量化入库→相似度检索→Prompt注入」的标准RAG流程。

### 3.4 记忆模块

双轨记忆机制：短期记忆保存最近20轮对话，长期画像持久化到本地JSON文件。

## 4. 核心数据流

用户输入 → 前端接收 → 业务层整合记忆上下文 → 路由分发 → 子Agent调用工具/知识库 → 生成回答 → 更新记忆 → 返回前端展示

## 5. 部署架构

单机部署模式，支持本地运行与Docker容器化部署两种方式，对外暴露8501端口提供Web服务。
